Codebox margin

<div>

<span class="sfe-codebox__marginp">

</div>

To emulate the gap of 1 tab or 4 normal spaces inside the codebox, an <html> element class of <sfe-codebox__marginp> was created to denote a spacing that appears to be like 4 normal keyboard typed spaces.

<css>

.sfe-codebox__marginp {

line-height: 1;

margin-top: 0px;

margin-bottom: 0px;

margin-left: 40px;

}

The <style> denotion of .sfe-codebox__marginp contains 40px of margin-left that is added to the inline spacing for the chosen line of code inside the codebox.

To implement this gap within the codebox, the html <span> tag is used. This is to conform to the inline spacing rules that other elements are sharing in this codebox, as opposed to using a block-level element such as <div>.

<style.css>

Media Queries

@media (max-width: 900px) {}

The contents of the webpage could be displayed clearly in an internet browser on a desktop computer. In contrast, the majority or all of the contents could be displayed just as clearly on a mobile device, which has a fraction of the pixel width of its counterpart.

Custom Properties

:root {

--ff-primary: 'Noto Sans', sans-serif;

--ff-secondary: 'Inconsolata', monospace;

--fw-reg: 300;

--fw-bold: 900;

}

Custom properties allow standardised modifications to <html>. <html> can be stylised using custom properties so that texts inside <body> and pages can be displayed correctly and uniformly. Alternatively, it provides a system to bring customized colours to html elements.

Pseudo-classes

:hover{}

:focus{}

Pseudo-classes like the ones above transform how a particular <html> element is displayed when it is selected as the centre of attention. The highlighting is done by styled classes inside the <html> element.

Flexbox

.logo__div {display: flex;}

Flex maintains a relative distance between the site logo and the home page logo even as the window resizes. The default function of the <css> property display flex is to align elements in a row.

.find-me__list {display: flex;}

When the viewer resizes their browser window, the above use of the flex display property keeps the relative distance between social icons. The distance between each social icon is not a user-defined metric value, but equivalent to a fraction of the space horizontally shared between icons across the width of the window.

Grid

.about-me__section {

display: grid;

grid-template-columns: 1fr 200px;

grid-template-areas:

"title title"

"subtitle image"

"body image";

grid-column-gap: 40px;

}

In the About me section, the grid property grid-template-columns is used to divide up the left-field into 1 fractional unit, and the right-field into 200px width for the profile picture. Using the grid property grid-template-areas, four distinct areas can be defined to specify the location of each content element. While the left partition can feature a top title area and bottom text area, a profile pic section on the right can span the height of the title and the text.

Avoiding confusing names in classList

Too many names used for different classes can confuse site-building. Using clear naming conventions in classes will assist html elements to indicate their intended action or function.

Succinct use of classes

Efficient naming conventions in the classList will assist in categorizing and creating a clear structure in <css>. Having a succinct classList will also emphasize usefulness in certain groups classes used by html elements or javascript.
